SQL Bookstore Sales Analysis Project

 Project Overview:

This project demonstrates SQL-based data analysis using a bookstore sales dataset. The database consists of three main tables: Books, Customers, and Orders, which simulate a real-world bookstore system. The goal of this project is to practice and showcase SQL skills such as data querying, aggregation, filtering, joins, and analytical queries.

The project answers multiple business questions related to sales performance, customer behavior, and book inventory management.

Dataset Description:
1. Books Table

Contains information about all books available in the store.

Columns

Book_ID – Unique identifier for each book

Title – Name of the book

Author – Author of the book

Genre – Category or genre of the book

Published_Year – Year the book was published

Price – Price of the book

Stock – Available stock quantity

2. Customers Table

Stores information about customers who purchase books.

Columns

Customer_ID – Unique identifier for each customer

Name – Customer's name

Email – Customer email address

Phone – Contact number

City – City of residence

Country – Country of residence

3. Orders Table

Contains information about customer purchases.

Columns

Order_ID – Unique identifier for each order

Customer_ID – ID of the customer placing the order

Book_ID – ID of the purchased book

Order_Date – Date when the order was placed

Quantity – Number of books ordered

Total_Amount – Total cost of the order

Key SQL Analysis Performed:

Some of the analytical tasks included in this project:

Retrieve all books from a specific genre

Find books published after a certain year

Calculate total sales revenue

Identify the most popular books based on orders

Find top customers by purchase amount

Analyze sales trends over time

Check stock availability and inventory insights

Join multiple tables to generate business insights
